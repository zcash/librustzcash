1//! Parser for [EIP-681](https://eips.ethereum.org/EIPS/eip-681) transaction requests.
2//!
3//! The top level of the parsing tree is the type [`EthereumTransactionRequest`].
4//!
5//! ## ABNF Syntax
6//!
7//! ```abnf
8//! request          = schema_prefix target_address [ "@" chain_id ] [ "/" function_name ] [ "?" parameters ]
9//! schema_prefix    = "ethereum" ":" [ "pay-" ]
10//! target_address   = ethereum_address
11//! chain_id         = 1*DIGIT
12//! function_name    = STRING
13//! ethereum_address = ( "0x" 40*HEXDIG ) / ENS_NAME
14//! parameters       = parameter *( "&" parameter )
15//! parameter        = key "=" value
16//! key              = "value" / "gas" / "gasLimit" / "gasPrice" / TYPE
17//! value            = number / ethereum_address / STRING
18//! number           = [ "-" / "+" ] *DIGIT [ "." 1*DIGIT ] [ ( "e" / "E" ) [ 1*DIGIT ] ]
19//! ```
20//!
21//! Where `TYPE` is a standard ABI type name, as defined in Ethereum Contract
22//! ABI specification. `STRING` is a URL-encoded unicode string of arbitrary
23//! length, where delimiters and the percentage symbol (%) are mandatorily
24//! hex-encoded with a % prefix.
25//!
26//! For the syntax of `ENS_NAME`, please consult
27//! [ERC-137](https://eips.ethereum.org/EIPS/eip-137) defining Ethereum Name Service.
28//!
29//! See
30//! [ABNF core rules](https://en.wikipedia.org/wiki/Augmented_Backus%E2%80%93Naur_form#Core_rules)
31//! for general information about the `ABNF` format.
32
33pub mod error;
34mod parse;
35pub use parse::{
36    Digits, EnsName, EthereumAbiTypeName, EthereumAddress, EthereumTransactionRequest, HexDigits,
37    Number, Parameter, Parameters, SchemaPrefix, UrlEncodedUnicodeString, Value,
38};
